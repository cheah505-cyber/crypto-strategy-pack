# 交易纪律与风控

## 10 条铁律

| # | 规则 | 状态 |
|---|------|------|
| 1 | **杜绝未来函数** — 信号计算不得使用决策点之后的信息 | ✅ `_preflight(df)` 强制检查 |
| 2 | **严防过拟合** — 参数必须通过样本外+敏感性分析 | ✅ 验证管线 Phase 3-4 |
| 3 | **计入真实成本** — 滑点/手续费/冲击成本/流动性限制 | ✅ 费用常量显式定义 |
| 4 | **剔除幸存者偏差** — 选股用历史成分股 | — 不适用（非选股策略） |
| 5 | **确保数据一致** — 实盘复权/清洗逻辑与回测完全一致 | 🔶 实盘部署时激活 |
| 6 | **环境高可用** — 服务器部署，断网断电自动恢复 | 🔶 实盘部署时激活 |
| 7 | **硬性风控** — 仓位上限/回撤熔断写死在代码中 | ✅ 策略代码常量定义 |
| 8 | **实时异常监控** — 心跳/成交率/滑点/净值异常报警 | 🟡 纸面交易已上线 |
| 9 | **严禁人工干预** — 正常回撤期内不得手动关停、改策略 | 🆕 行为纪律 |
| 10 | **识别结构变化** — 回撤突破历史极值或市场规则根本性改变时暂停 | 🆕 行为纪律 |

## 严禁人工干预

策略触发风控熔断前，人类角色是观察者，不是操作者：

- 浮亏在历史最大回撤范围内 → **不干预**。这是回测已见过的正常波动。
- 连续亏损未达熔断线 → **不干预**。胜率不是 100%，连续亏是概率的兑现。
- 主观觉得"这次不一样" → **不干预**。每次恐慌都像不一样，事后看都一样。
- 选择性跟单（手动跳过某些信号）→ **禁止**。跳过亏损信号 = 事后诸葛亮。

唯一允许人工介入的情况：策略触发代码中写死的熔断/强平线。

## 识别结构变化

以下任一发生时，暂停交易并重新评估：
- 回撤突破历史极值且持续扩大
- 交易所规则根本性改变（下架、杠杆调整）
- 底层资产发生结构性断裂（硬分叉、监管定性、协议被攻击）
- 策略核心假设被证伪

暂停 → 回测验证 → 决策（继续/调整/退役）。不抄底、不平摊、不死扛。

## 风控参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 每笔风险 | 10% | 单笔最大亏损占权益比例 |
| 杠杆上限 | 10x | Binance 永续合约杠杆上限 |
| 熔断 | 5 连亏 → 停 24 bars | 匹配 42% 胜率的统计概率 |
| 波动率退出 | ATR% > 3.5% | P97 极端波动主动平仓 |
| 趋势止损 cap | 12% | 防止极端滑点下无限亏损 |
| 过渡止损 cap | 5% | 过渡模式紧止损上限 |

## 回测安全规则

### Preflight 检查（代码硬门禁）

每个 `run_backtest()` 入口必须调用 `_preflight(df)`，不通过则回测不执行：

- 信号列和指标列存在性验证
- DatetimeIndex 严格单调递增、无重复
- 预热期后信号列零 NaN
- 信号列类型为布尔/整数
- OHLC 逻辑一致性

### Sanity Test

每个 `run_backtest()` 前必须执行 `_run_sanity(df)`：
- 强制单一入场持有至结束 → PnL = 价格变化减成本
- 全部信号关闭 → 零交易、零收益
- 固定时间点单笔交易 → PnL 与手动计算一致

### 禁止模式

- `df["close"].rolling(N).max()` 直接比较无 `shift(1)`
- 缺失值用全局统计量填充（如 `fillna(df.mean())`）
- 用当前 bar 的 high/low 决定当前 bar 突破信号
