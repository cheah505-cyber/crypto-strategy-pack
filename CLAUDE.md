# 加密货币量化交易

> **状态**: 初始化阶段，以下为规划中的技术栈和结构。

## 技术栈（规划）
- Python 3.12，pandas, numpy, scikit-learn, PyTorch
- 交易所接入：CCXT
- 数据库：SQLite（本地）/ PostgreSQL（生产候选）
- 可视化：matplotlib, plotly

## 回测纪律
见全局规则 `backtest-safety.md`：时间点对齐、前视偏差检查、时间顺序分割、参数记录、样本外独立验证。不在此重复。

## CCXT 使用约定
- 统一交易所 API（统一返回值格式）
- 订单操作带重试机制和错误处理
- 测试网和主网配置分离

## 策略开发流程

### 先头脑风暴
开始写任何策略代码前，必须：
1. 明确策略假设和逻辑
2. 提出 2-3 种实现方案并讨论权衡
3. 达成一致后再动手

### 实施流程
1. 产生想法 → 记录到 Obsidian
2. 因子验证（IC 分析、分层回测）
3. 策略回测（样本内 + 样本外）
4. 参数敏感性分析
5. 模拟交易
6. 记录结果到 Obsidian
