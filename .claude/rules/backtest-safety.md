# 回测安全规则

涉及回测代码时自动启用。

## 时间点对齐
- 每次回测前显式验证时间点对齐
- 特征矩阵中任何一行的数据不得包含该行时间点之后的信息
- 使用 `shift()` 等操作确保未来数据不泄露

## 数据分割
- 训练集、验证集、测试集严格按时间顺序分割
- 禁止随机分割时间序列数据
- 留出足够长的回测样本外区间（至少占总时长的 20%）

## 前视偏差检查清单
- [ ] 价格数据是否使用了未来信息（如 close 当天的 high/low）
- [ ] 信号生成是否使用了未来数据点
- [ ] 因子计算中是否有跨时间点的泄露
- [ ] 重采样时是否混入了未来数据
- [ ] 缺失值填充是否使用了未来统计量

## 过拟合防护
- 参数优化后必须在完全独立的样本外数据上验证
- 记录每次回测的参数组合、随机种子、数据范围
- 多策略比较时使用相同的样本外区间
- 警惕"过度优化"——参数对小幅变化应相对稳定

## 费用和滑点

回测必须计入全部交易成本，严禁零成本假设：

- **手续费** — maker/taker 费率按交易所实际值设置，区分开仓和平仓
- **滑点** — 按币种流动性和时框合理估算，大盘平稳期 ≥ 0.02%，极端行情 ≥ 0.1%
- **冲击成本** — 大仓位按成交量比例估算价格冲击，不得假设无限深度
- **资金费率** — 永续合约必须计入每 bar 资金费率
- **涨跌停/流动性限制** — 极端波动时可能无法以信号价格成交，需保守估算执行价

每次回测报告中显式声明使用的费用参数。费用参数变动视为策略变更，需重新跑全验证管线。

## 自动化优化循环

当跑参数扫描/自动调参时，以下规则防止自欺和过度优化：

### 一次一变
每次实验只改一个变量。同时动止损、仓位、因子权重 = 不知道谁起作用。结果不可归因。

### 评估器不可变
回测脚本是基准值。Agent 绝不能修改评估器——改了就等于换尺子，所有对比失效。硬门禁。

### 策略升级节奏
- 1-5 轮：低垂果实（明显可改的小优化）
- 6-15 轮：系统性探索（每个参数逐个遍历）
- 16-30 轮：结构性改动（算法替换、新因子引入）
- 30+ 轮：激进实验（完全不同思路）
- 20+ 轮无提升 → 更新策略文档，不再在同一方向浪费算力

### 崩溃处理
- 笔误/缺 import → 修，重新跑
- 思路本身有问题 → 回退，记录 "crash"，跳过
- **5 次连续 crash → 暂停，告警。不继续烧算力**

### 每 10 轮复盘
分析 results 找规律（如"缓存类改动稳定提升 5-10%"，"重构类从未提升指标"），更新策略文档。后续轮次受益于积累的知识。

### 简洁优于复杂
小提升但引入丑陋复杂度 → 不值。同等性能但代码更简单 → 是赢。删掉代码但保持效果 → 最好结果。

### 超时
评估运行超过 2.5× 预期时间 → 杀掉视为 crash。可能是死循环或参数不可行。

### 蒙特卡洛验证
- 用 **bootstrap**（有放回重采样），不用 shuffle（打乱顺序结果不变——乘法交换律）
- 报告 P1/P5/P50/P95 收益分布，不只看均值

### Equity 计算检查
- **离谱数字先疑代码** — 总收益 >1000%、夏普 >10 先假设有 bug
- 常见陷阱：MTM 估值 + realized return 双重计算、全仓复利 on 部分仓位
- 检验方法：手动挑 3 笔交易，用计算器复算 equity 变化

### 回测 Sanity Test
每个新回测脚本必须含至少一个 sanity test：
- 永远持仓 → 收益应等于 benchmark
- 永远空仓 → 收益应为 0（减手续费）
- 固定单笔交易 → 手动可验 PnL

## 回测报告完整性
每个回测结果包含：
- 数据范围和时间粒度
- 总收益率、年化收益率、最大回撤
- 夏普比率、卡玛比率
- 交易次数和胜率
- 对比基准的表现
