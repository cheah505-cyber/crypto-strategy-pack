# 策略验证管线

涉及新策略开发或回测结果判断时自动启用。

## 8 阶段验证

新策略必须逐阶段通过，每阶段 git commit 存档：

| Phase | 内容 | 门禁 |
|---|---|---|
| 0. 数据 | CCXT 拉取 + DQS | 行数 ≥ 5000, DQS ≥ 85 |
| 1. 因子 IC | ≥3 因子 × ≥3 参数 | IC/ICIR/p/效应量全部报告，前视偏差检查通过 |
| 2. 策略回测 | 完整回测 | 7 项指标完整，交易 ≥ 10 笔 |
| 3. 参数稳定性 | ≥1 参数 × ≥3 档位 | 全部档位盈利，全部 PF > 1 |
| 4. 样本外 | 时间分割 Train/Test | 测试期收益 > 0，夏普 ≥ 训练期 50% |
| 5. 蒙特卡洛 | Bootstrap ≥ 1000 次 | P5 收益 > 0 |
| 6. 极端行情 | ≥3 崩盘窗口 | 强平/最大窗口回撤报告 |
| 7. 跨币 | ≥1 其他币种 | 同策略验证 |
| 8. 多时框 | ≥1 其他时框 | 同策略验证 |

## 决策标准

回测完成后，按以下框架判断，不以单一固定分数线做决策：

**通过条件（全部满足 → ✅ 合格）：**
- 样本外收益为正
- 全面验证管线（8 阶段）全部门禁通过
- 无 fatal 缺陷（强平、Sanity Test 失败、前视偏差）

**改进条件（满足任一 → 🔁 继续优化）：**
- 样本外收益为正但夏普/回撤明显弱于样本内（保留率 < 50%）
- 显著未跑赢基准（需论证策略仍有存在价值）
- 参数敏感性测试中存在不稳定的参数档位

**放弃条件（满足任一 → ❌ 废案）：**
- 样本外收益为负
- 交易次数不足以做统计推断（参考 < 10 笔）
- 3 次改进后核心指标无实质提升

每轮决策必须记录依据：观测数据 + 判断理由 + 下一步行动。
