# Crypto 量化经验（论证阶段）

## 方法论

**IC 分析先于回测。** 先验证因子有预测力，再写策略代码。7 因子 22 组合 30 分钟跑完，省了写无效策略的时间。

**假设随时推翻，不自证预设。** 定了趋势跟踪方向，数据说均值回归，立刻转向。方向错不丢人，坚持错才丢人。

**先校准再投入，用最小成本验证可行性。** 跑一次 IC 分析三分钟发现方向，比花两天写策略代码值。

## 风控

**全仓回测数字不可信。** 无风控版 +6435% 和 +127% 都是真的，但分别来自复利 bug 和无止损。加风控后 +403% 夏普 1.75 才是诚实数字。

**熔断阈值要匹配胜率。** 44% 胜率下 3 连亏概率 16.6%，每 6 笔触发一次。3L→48bar 熔断把策略锁死到仅 9 笔/3.4年。5L→24bar 才合理。

**做空不是锦上添花，是必要组件。** 均值回归信号对称，现货只能吃到超卖反弹，永续做空多吃 42 笔 = 总收益跳 4 倍。

## 验证

**蒙特卡洛必须 bootstrap，不能 shuffle。** 乘法交换律下打乱顺序结果永远相同。有放回重采样才是真正的模拟。

**跨币是照妖镜，多时框是过滤网。** ETH +403% 不代表 BTC 也能。策略只在 4h 有效——1h 噪音大，日线信号少。

**先找跨框架信号过滤，再深挖单框架参数。** 1h 独立夏普 1.4，4h 独立夏普 1.3。加一个简单的"仅 4h ADX>30 时交易"→牛市夏普 1.73，回撤 -25%。单行过滤代码胜过百轮参数优化。框架间的信息差异 > 框架内的参数空间。

**OOS 夏普衰减先疑训练数据不具代表性。** 原来 0.60→0.35 是纯牛市训练的假象。加入熊市数据后 IS 0.82→OOS 0.85（保留率 103%）。数据覆盖不足比过拟合更像罪魁祸首。

**Walk-Forward 短数据会误判。** 4/7 窗口正（2023-2026）→ 17/26（2019-2026）。窗口数要够才能下结论。

**Walk-Forward 最难过，样本外次之。** 连续 7 窗口 OOS 比单次 split 严格得多。4/7 PASS 是边缘，但累积 OOS +49%。

**极端行情测试比夏普更重要。** 5 崩盘窗口 3 次对冲 B&H 损失，0 强平——这才是实盘真正要命的场景。

## 工程

**命名即文档。** `adx_perp.py` → `adx_adaptive_perp_eth_4h.py`。文件名说明策略+对象+时框。

**定方案就是关键节点，架构决策必须写入记忆。** 自建引擎方案确立后立即写入 project memory，避免下次对话从头摸索。

**零成本纸面交易管线 = GitHub Actions + git 持久化。** 信号+纸面交易脚本部署到 Actions 定时运行，状态文件（JSON/CSV）由 CI 自动 commit+push 回 repo，跨运行保持仓位连续性。无需 VPS，不开电脑，手机收 Telegram。比本地跑多了 git 状态的额外好处——每笔交易有 immutable record。

**牛市只看牛市是调参陷阱。** 纯牛市数据校准 ATR 得到 1.20x 推荐，全周期验证才发现 2.5x 才是最优。bull-only 校准在熊市崩盘。调参必须覆盖完整的牛熊周期。

**ADX 区间阈值比趋势阈值敏感得多。** RANGE=15→20 的 5 个点变化让 Sharpe 从 0.46 跳到 1.303（+184%）。TREND 的变化影响小得多。先调区间阈值。

**ATR 百分比化是无量纲修复的第一步。** ATR 用原始价格 → 止损在 % 项下漂移（CV=0.38）。改为 `ATR/close*100` 后数学恒等但语义正确——止损 cap、跨品种比较、因子标准化都有了统一基准。改前改后回测必须恒等，否则有 bug。

**止损 % cap 不可压在 P95 上。** 8% 趋势 cap 刚好卡在 ATR% P95（2.5×ATR%=7.98%）→ 5% 的交易止损被截断 → Sharpe 从 1.184 跌到 0.891（-25%）。cap 位置至少应落在 P99 以上（趋势 ~12%）或基于 RISK_PER_TRADE 留足安全边际。

**波动率退出是不对称的防御。** ATR% > 3.5% (P97) 主动平仓 → 83% 优于持有，均收益 +9.75%。盈利退出锁利（+15.7% vs +7.5%），亏损退出浅止损（-2.2% vs -5.4%）。少数情况过早下车（1/24 错过 +73%），但累计净收益 +157pp。这是边缘优化不是结构修复——DD 仅降 2pp。
