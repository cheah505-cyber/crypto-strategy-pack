# Crypto 量化经验

> 论证阶段（2026-05-22→2026-06-06）全部经验总结。

## 方法论

- **IC 分析先于回测** — 先验证因子有预测力，再写策略代码。7 因子 22 组合 30 分钟跑完。
- **假设随时推翻，不自证预设** — 定了趋势跟踪方向，数据说均值回归，立刻转向。
- **先校准再投入，用最小成本验证可行性** — 跑一次 IC 分析三分钟发现方向，比花两天写策略代码值。

## 风控

- **全仓回测数字不可信** — 无风控版 +6,435% 和 +127% 都是真的，但分别来自复利 bug 和无止损。加风控后 +403% 夏普 1.75 才是诚实数字。
- **熔断阈值要匹配胜率** — 44% 胜率下 3 连亏概率 16.6%。5L→24bar 熔断才合理。
- **做空不是锦上添花，是必要组件** — 均值回归信号对称，永续做空多吃 42 笔 = 总收益跳 4 倍。

## 验证

- **蒙特卡洛必须 bootstrap，不能 shuffle** — 乘法交换律下打乱顺序结果永远相同。有放回重采样才是真正的模拟。
- **跨币是照妖镜，多时框是过滤网** — ETH +403% 不代表 BTC 也能。策略只在 4h 有效。
- **先找跨框架信号过滤，再深挖单框架参数** — 1h 独立夏普 1.4，4h 独立夏普 1.3。加"仅 4h ADX>30 时交易"→ 牛市夏普 1.73，回撤 -25%。单行过滤代码胜过百轮参数优化。
- **OOS 夏普衰减先疑训练数据不具代表性** — 纯牛市训练 Sharpe 0.60→0.35。加入熊市数据后 IS 0.82→OOS 0.85（保留率 103%）。数据覆盖不足比过拟合更像罪魁祸首。
- **Walk-Forward 短数据会误判** — 4/7 窗口正 → 17/26 窗口正。窗口数要够才能下结论。
- **Walk-Forward 最难过，样本外次之** — 连续 7 窗口 OOS 比单次 split 严格得多。
- **极端行情测试比夏普更重要** — 5 崩盘窗口 3 次对冲 B&H 损失，0 强平。

## 工程

- **命名即文档** — 文件名说明策略+对象+时框。
- **定方案就是关键节点，架构决策必须写入记忆** — 自建引擎方案确立后立即记录。
- **零成本纸面交易管线 = GitHub Actions + git 持久化** — 信号+纸面交易脚本部署到 Actions 定时运行，状态文件由 CI 自动 commit+push 回 repo。
- **牛市只看牛市是调参陷阱** — 纯牛市校准 ATR 得到 1.20x，全周期验证发现 2.5x 才是最优。
- **ADX 区间阈值比趋势阈值敏感得多** — RANGE=15→20 的 5 个点变化让 Sharpe 从 0.46 跳到 1.303（+184%）。
- **ATR 百分比化是无量纲修复的第一步** — ATR/close*100 后止损 cap、跨品种比较、因子标准化都有了统一基准。
- **止损 % cap 不可压在 P95 上** — 8% 趋势 cap 刚好卡在 P95，5% 的交易止损被截断 → Sharpe 从 1.184 跌到 0.891（-25%）。
- **波动率退出是不对称的防御** — ATR% > 3.5% (P97) 主动平仓 → 83% 优于持有。盈利退出锁利（+15.7%），亏损退出浅止损（-2.2%）。
